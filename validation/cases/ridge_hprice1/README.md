# Ridge regression on Wooldridge `hprice1`

This validation case estimates a Ridge regression of log house price on log lot size, log square footage, number of bedrooms and a colonial dummy using the Wooldridge `hprice1` dataset.

## Model

```
lprice ~ llotsize + lsqrft + bdrms + colonial
```

## Dataset

- **Name:** `wooldridge::hprice1`
- **Source:** R package `wooldridge`; also available in Python via `wooldridge`.
- **Licence:** Public teaching dataset.
- **Size:** 88 observations × 11 variables (log-transformed variables generated by the reference scripts).

## Reference implementation

- **Python:** primary reference implementation.
- **R:** a reference script using `MASS::lm.ridge` is provided and runs successfully. The validation runner prefers the Python reference when both are available, so the R reference is available and would be used if the Python reference is unavailable or if the runner is configured to use R.
- **Hayashi:** `ridge(lprice ~ llotsize + lsqrft + bdrms + colonial, df, alpha=0.1)`

## Compared quantities

- coefficients only (Ridge/ElasticNet have no analytical standard errors in these implementations)

## Tolerances

| Quantity | Tolerance | Rationale |
|---|---|---|
| coefficients | 1e-1 | Different implementations standardise X differently and handle the intercept differently; Python is aligned to Hayashi's closed-form formula |
