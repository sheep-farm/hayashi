# ARIMA(1,1,0) on a simulated random walk

This validation case estimates an ARIMA(1,1,0) model on a simulated random walk.

## Status

`active`

## Model

```
Δ rw_t = c + φ Δ rw_{t-1} + ε_t
```

with `rw_t` a pure random walk (`φ = 0` in population).

## Dataset

- **Name:** `simulated_rw`
- **Source:** generated by `validation/cases/arima_book/data/gen.hay`
- **Size:** 500 observations
- **Columns:** `rw` only

The data are generated with `seed(42)` as:

```hay
seed(42)
input df
id
1
end
for i in 1..=9 { df = append(df, df) }
df |> filter(_n <= 500)
generate df e  = rnormal()
generate df rw = cumsum(e)
```

## Reference implementation

- **R:** `arima(df$rw, order = c(1, 1, 0), method = "CSS-ML")`
- **Python:** `statsmodels.tsa.statespace.SARIMAX(df["rw"], order=(1, 1, 0), trend="n")`
- **Hayashi:** `arima(df, rw, p=1, d=1, q=0)`

## Compared quantities

- `coefficients` — only `ar.L1`
- `standard_errors` — only `ar.L1`

The intercept reported by Hayashi is **not** compared because the R and Python references are estimated without trend (`trend='n'` / no intercept in the differenced equation).

## Tolerances

| Quantity | Tolerance | Rationale |
|---|---|---|
| coefficients | 1e-2 | ARIMA estimates on a random walk should be close to zero |
| standard_errors | 5e-2 | Slightly looser for cross-package SE approximation |
