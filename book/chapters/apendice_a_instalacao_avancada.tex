\section{Compilação com suporte a ODBC}

Para conectar \hayashi{} a bancos de dados via ODBC (SQL Server, Oracle, etc.):

\begin{lstlisting}[style=inline, language=sh]
cargo install hayashi-lang --features odbc
\end{lstlisting}

Requer driver ODBC instalado no sistema (\texttt{unixODBC} no Linux).

\subsection*{Modelos econométricos avançados}

Os estimadores abaixo requerem a crate \texttt{greeners} como dependência
do interpretador. Em builds de desenvolvimento (\texttt{cargo build}), ela é
compilada automaticamente. Nenhuma instalação adicional é necessária.

\begin{tabular}{ll}
\toprule

                     \texttt{mundlak}, \texttt{xtgls}, \texttt{mixedlm}, \texttt{clogit} \\

\subsection*{Uso da memória}

Modelos computacionalmente intensivos podem requerer ajuste de stack:

\begin{lstlisting}[style=inline, language=sh]
RUST_MIN_STACK=8388608 hay script.hay  // 8 MB de stack
\end{lstlisting}

Relevante para VAR/SVAR com muitas defasagens ou PCA em datasets amplos.

\section{Integração com editores}

\subsection*{Neovim}

Adicione ao \texttt{init.lua} para reconhecimento de sintaxe básico:

\begin{lstlisting}[style=inline, language=Hayashi]
vim.filetype.add({ extension = { hay = "hayashi" } })
\end{lstlisting}

Highlighting completo via Tree-sitter está no roadmap.

\subsection*{VS Code}

Extensão disponível na marketplace como \texttt{hayashi-lang}.

\section{Compilação de desenvolvimento}

Para iterar com builds locais (sem \texttt{--release}):

\begin{lstlisting}[style=inline, language=sh]
git clone https://github.com/sheep-farm/hayashi
cd hayashi
cargo build            // debug build (recomendado para dev)
./target/debug/hay script.hay
\end{lstlisting}

O build debug tem verificações extras de bounds e mensagens de erro detalhadas.
Não usar \texttt{--release} em desenvolvimento — o ganho de velocidade não
compensa a perda de diagnóstico.

\subsection*{Execução de scripts}

\begin{lstlisting}[style=inline, language=sh]
./target/debug/hay script.hay        // executa script
./target/debug/hay                   // abre REPL interativo
./target/debug/hay < script.hay      // lê do stdin
\end{lstlisting}
