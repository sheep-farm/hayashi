## Methodology

- Synthetic datasets generated by `benchmarks/datasets/generate.py`.
- Test sizes: `n = 1,000` and `n = 10,000`.
- Repetitions: 5, with the first discarded as warmup.
- Time measured by the Python orchestrator (`time.perf_counter`).
- Memory measured by polling `/proc/<pid>/status` (`VmRSS`) of the main
  process and child processes.
- Hayashi compiled in release mode (`target/release/hay`).

## Interpretation / caveats

- Hayashi is consistently faster and uses less memory on this machine.
- Differences of up to 100x appear in MLE-optimized estimators (ARIMA,
  GARCH), where Hayashi seems to converge quickly.
- Python and R compute more statistics by default (robust covariance,
  diagnostics, influence). This benchmark measures the default command
  time, not a minimally equivalent implementation.
- Hayashi memory ranges from 0.1 MB to 12.7 MB, while R/Python often
  allocate 80–280 MB. Part of this is due to loading large numerical
  libraries in the child process.
- Results may vary on other machines, operating systems, compilation flags,
  or dependency versions.
